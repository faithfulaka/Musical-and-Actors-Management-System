Homework 1: Tasks for fane1

Your personal task is to develop a system for musicals and their actors.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Actor. With the following private fields:

  1. String name
  2. String training
  3. String type
  4. int awards
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Musical. With the following private fields:

  1. int id
  2. String name
  3. String composer
  4. List<Actor> actors
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Musical> musicals in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one musical with two actors. [4 marks]

Create controller co2103.hw1.controller.MusicalController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/musicals" should show view musicals/list. [2 marks]
    • Add the list of all musicals from co2103.hw1.Hw1Application to the model as attribute musicals. [2 marks]
    • Create the corresponding JSP to list the name and composer of each musical followed by a link to URL "/actors?musical={id of musical}", e.g., "/actors?musical=3". [4 marks]
  2. Requests for URL "/newMusical" should show view musicals/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="musical" and action="/addMusical" and inputs for all fields of class Musical except for actors. [5 marks]
  3. POST requests for URL "/addMusical" should add a new musical to the list of all musicals in co2103.hw1.Hw1Application and redirect to /musicals. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.ActorController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/actors" should show view actors/list. [2 marks]
    • Based on request parameter musical determine the corresponding musical from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all actors of that musical to the model as attribute actors. [2 marks]
    • Create the corresponding JSP for the view to list the name, training, type, and awards of each actor. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newActor?musical={id of musical whose actors are listed}", e.g., "/newActor?musical=3". [4 marks]
  2. Requests for URL "/newActor" should show view actors/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="actor" and action="/addActor" and inputs for all fields of class Actor. [6 marks]
    • Also add the request parameter musical with the ID of the current musical. [4 marks]
  3. POST requests for URL "/addActor" should add a new actor to the list of actors of the musical identified by request parameter musical and then redirect to /musicals. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Musical and make it available in controller MusicalController. [4 marks]

When adding a new musical validate that

  1. The value of field id is unique within the list Hw1Application.musicals. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field composer is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Actor and make it available in controller ActorController. [4 marks]

When adding a new actor validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field training is not empty. [2 marks]
  3. The value of field type is either "Soprano", "Mezzo soprano", or "Contralto". [3 marks]
  4. The value of field awards is between (including) 0 and 7. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
